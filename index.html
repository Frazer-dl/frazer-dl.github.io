Архитектура

Программа состоит из 3 обособленных модулей.
1 - HDM (hart driver module) - драйвер подключения к ком-порту, отвечает за обмен данными между устройствами и ПК и за автоматический поиск устройств и сохранение в БД, также на это уровне располагаются две базы данных(для трендов и устройств).
2 - AM (api module) - модуль апи для координации и обработки различных запросов от интерфейса пользователя.
3 - WM (web module) - модуль интерфейса пользователя.
Структура базы данных состоит из трёх основных таблиц и одной вспомогательной.
1 - Device - служит для сохранения данных найденых устройств.
2 - Parameter - служит для хранения значений переменных.
3 - DeviceInfo - служит для хранения информации о приборе

Идеи

Аналитика отказов датчиков

Сбор информации по отказам и формирование различных отчётов и рекомендаций в автоматическом режиме

назад
Система составления плановых графиков обслуживания

Внедрение системы учёта и планирования текущего, капитального, а также межремонтного обслуживания

назад
Учёт приборов

EAM система для учёта комплектущих, составления закупочных ведомостей. Учёт и планирование оборудования на срок эксплутации объекта

назад
Интеграция в SCADA системы

По средствам OPC обработка и предоставление различных данных, в том числе аварийных сообщений с датчиков

назад
Взаимодействие с системой с экрана планшета/телефона

Создание гибкой, портативной системы для простого взаимодействия по средствам беспроводной связи с телефонов и планшетов

назад
Создание эмулятора HART устройств

Данное решение будет служить наглядным примером демонстрации функций системы, а также для мониторинга, диагностики и нагрузочного тестирования программы

назад
Интеграция HART 7

Нужно идти в ногу со временеи и поддерживать новые версии протокола

назад
Интеграция DD файлов

Данный алгоритм позволит универсально использовать программу с любым прибором

назад
